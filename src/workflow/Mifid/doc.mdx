## Description

The Investor Preferences workflow demonstrates how you can create a more personalized experience for investors by defining settings in the Screener and Portfolio Analysis X-Ray APIs to:

•&nbsp;&nbsp;Create a set of filters based on an investor's preferences that you use to search Morningstar's database for matching investments.<br />
•&nbsp;&nbsp;Use these filters to flag investments that match the investor’s preferences.

These capabilities can be used to:

•&nbsp;&nbsp;Assess an investor's sustainable investing preferences using a questionnaire. <br />
•&nbsp;&nbsp;Apply a flag to securities that align with the investor’s preferences, making them easily identifiable for use in, for example, a list of results in a fund screener.<br />
•&nbsp;&nbsp;Generate a client portfolio overview that includes an insight into the portion of the portfolio that is aligned with the investor's preferences.<br />

Investor Preferences capabilities are also supported by the Security Details API.

### Request Payload Example

| Query Parameter        | Description                                                                   | Example                                                                          |
| :--------------------- | :---------------------------------------------------------------------------- | :------------------------------------------------------------------------------- |
| `languageId`           | ISO culture codes. If not provided, defaults to language defined in settings. | "languageId": "en—GB"                                                         	|
| `currencyId`           | ISO culture code. If not provided, defaults to language defined in settings.  | "currencyId": "GBP"                                                            	|
| `universeIds`          | A pipe-delimited string of unique identifers of a universe.                   | "universeIds":"FOGBR$$ALL&#124;E0EXG$XLON"                             			|
| `filterDataPoints`     | Comma-separated list of filter definitions to be returned.                    | "filterDataPoints":"CustomCategoryId&#124;CustomCategoryId2&#124"				|
| `filters`              | A list of filter constraints separated by a pipe-delimited string.            | "filters":"BrandingCompanyId:EQ:BN00000JKZ&#124;Yield_M12:BTW:2:4"				|
| `page`                 | Defines the number of output pages.                                           | "page": "1"                                                                      |
| `pageSize`             | Defines the number of rows per page.                                          | "pageSize": "20"                                                                 |
| `sortOrder`            | Defines the data point on which to order the results and the order direction. | "sortOrder": "name asc"                                                          |
| `securityDataPoints`   | Comma or pipe separated list of data points to return.                        | "securityDataPoints": "SecId&#124;LegalName&#124;DomicileId&#124;"     			|
| `calculatedDataPoints` | Object holding the list of data points to calculate and return boolean.       | `Please See Below`                                                               |
| `term`                 | String to use to search for securities such as names, identifiers or symbols. | "term":"LINXEA"                                                                  |

### Request Example

Requests to the API have the following format:

```javascript dark
https://www.emea-api.morningstar.com/ecint/v2/xray?outputtype=json&languageId={{languageId}}&portfolioDataPoints={{portfolioDataPoints}}&holdingDataPoints={{holdingDataPoints}}&benchmarkDataPoints={{benchmarkDataPoints}}
```

### Request Payload Example

| Query Parameter | Description                                                                   | Example            |
| :-------------- | :---------------------------------------------------------------------------- | :----------------- |
| `languageId` |	ISO culture codes. If not provided, defaults to language defined in settings. |`languageId=en-GB` |
| `portfolioDataPoints` |Comma-separated list of portfolio data points.|`portfolioDataPoints=StyleBox,GlobalStockSector` | 
| `holdingDataPoints` |Comma-separated list of holding data points.|`HoldingDataPoints=ShowBreakdown` |
| `benchmarkDataPoints` | Comma-separated list of benchmark data points.|`benchmarkDataPoints=GrowthSeries,HistoricalPerformanceSeries,ShowStatistic` |
